**ステータス**: 構想・着手保留
**位置づけ**: TD廃止 → 次フェーズの主要新機能
**起点メモリ**: `reference_prism_gacha_repo.md` 「将来案: 派閥対戦オートバトラー (2026-04-26 採用方針確定)」

---

## 1. コンセプト

ガチャで集めたキャラ5人で編成し、 **観戦型の自動バトル**を楽しむミニゲーム。 同派閥3人以上でシナジー、 デイリーチャレンジで継続性。 RPG経験値や課金要素は持たず、 **キャラ成長履歴の解放**を主報酬とする。

- **Auto-battler**: プレイヤーは編成だけ決めて、 戦闘はAI同士の自動進行
- **観戦のみ**: 戦闘中の操作なし、 早送り/巻き戻し対応
- **CPU専用 (初期)**: 他プレイヤー対戦は将来、 まずは固定CPUに勝ち抜く形
- **報酬**: 対戦経験値で キャラ desc / 凸物語の追加解放 (キャラ成長履歴 #13 と統合)

## 3. データモデル

### 3.1 キャラステータス (POOL に追加)
既存 POOL の各キャラに以下を追加:
```js
{
  ...既存フィールド,
  arena: {
    hp: 1200, atk: 180, def: 120, spd: 95,
    mp: 100, mpRegen: 8,
    skills: [
      { id: 'flame_slash', name: '焔斬', cost: 30, dmg: 240, ct: 3 },
      { id: 'rage', name: '激昂', passive: true, condition: 'hp<50%', effect: 'atk+30%' },
    ],
    ult: { id: 'phoenix_rise', name: '不死鳥の復活', cost: 100, dmg: 600, special: 'revive_self_once' },
  },
}
```

tier 別目安 (バランス調整用テンプレ):
| tier | HP    | ATK  | スキル数 | 必殺技 |
|------|-------|------|----------|--------|
| R    | 600   | 80   | 1        | なし   |
| SR   | 800   | 110  | 1        | なし   |
| SSR  | 1000  | 150  | 2        | なし   |
| UR   | 1300  | 200  | 2        | あり   |
| LR   | 1800  | 280  | 3        | あり (固有) |

### 3.2 派閥シナジー定義
```js
const FACTION_SYNERGIES = {
  church:   { 3: 'def+10%', 4: 'def+20%+heal8%', 5: 'def+35%+heal15%+リジェネ' },
  aquasis:  { 3: 'mp+20',   4: 'mp+40+回避5%', 5: 'mp+60+回避15%+反射' },
  crimson:  { 3: 'atk+10%', 4: 'atk+20%+クリ10%', 5: 'atk+35%+クリ25%+貫通' },
  // 全14派閥分 (5人未満の小派閥は3人シナジーまで)
};
```

### 3.3 戦闘ログ (リプレイ用)
```js
{
  battleId: 'b_20260501_xxxxx',
  date: 1730000000000,
  team: ['プリズマ','イザベル',...],
  enemy: 'cpu_basic_lv5',
```

## 4. 戦闘アルゴリズム (シミュレーション)

### 4.1 ターン進行
1. 全キャラの SPD 順にイニシアチブ計算
2. 最大 SPD のキャラから1ターン1アクション
3. アクション選択優先順位:
   - 必殺技 cost 充足 + cooldown 0 → 必殺技
   - スキル cost 充足 + cooldown 0 → スキル (複数あれば dmg/heal の高い方)
   - それ以外 → 通常攻撃
4. ターゲット選定: 通常 = 敵で HP最少、 スキル/必殺技 = スキル種別による

### 4.2 ダメージ計算
```
damage = (atk * skill_multiplier - target.def * 0.5) * crit_factor * faction_synergy_atk
crit_factor = (random < crit_rate) ? 1.5 : 1.0
最低保証 = 1
```

### 4.3 勝敗判定
- 全キャラ HP=0 で敗北
- 30ターン経過で時間切れ → 残HP合計が多い側の勝ち
- 状態異常持ち越しなし (1戦完結)

## 5. CPU 対戦相手 (Phase 1)

固定 5レベル + 派閥特化型を用意:
- Lv1-3: ランダムR/SR編成 (チュートリアル相当)
- Lv4-5: 派閥シナジー有 (church 3人, crimson 3人 等)
- Lv6-8: SSR混在
- Lv9-10: UR持ち
- Boss-1: 全派閥混在 LR持ち (週1チャレンジ)
- Boss-2: 同派閥5人特化 (派閥特化討伐)

CPU編成は固定でも、 順番のランダム性とサイコロ結果で毎回変わる。

## 7. 技術スタック

- **フロントのみ完結**: 既存スタック (Vanilla JS + Firebase Realtime DB)
- **戦闘ロジック**: 純粋関数 `simulateBattle(team, enemy, seed): BattleLog` で書く → リプレイ可能
- **シード**: 戦闘開始時の Date.now() を seed に Mulberry32 PRNG を使い、 同じ team+enemy+seed なら同じ結果になる(リプレイ用)
- **新ファイル**: `arena.js` (戦闘ロジック+UI)、 `arena/data.js` (キャラステータス+敵編成テーブル)
